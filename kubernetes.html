Kubernetes Deployment

Replace your LiteLLM Deployment + ConfigMap with AgentGateway. Two paths: raw manifests or Helm + Gateway API.

Resource Comparison

AgentGateway is dramatically lighter than LiteLLM in Kubernetes.

Resource LiteLLM AgentGateway
Container imagedocker.litellm.ai/berriai/litellm:main-stablecr.agentgateway.dev/agentgateway:latest
CPU request2 cores250m
Memory request4 Gi128 Mi
CPU limit4 cores1 core
Memory limit8 Gi512 Mi
Readiness probe delay120s5s
RuntimePythonRust (static binary)
Image size~1.5 GB~50 MB
16x less memory, 24x faster startup These aren't theoretical numbers. Rust binaries don't need a Python runtime, package manager, or virtual environment in the container.

Option 1: Raw Kubernetes Manifests

Direct translation of your existing LiteLLM manifests to AgentGateway.

ConfigMap

✕ LiteLLM ConfigMap
apiVersion: v1
kind: ConfigMap
metadata:
  name: litellm-config
data:
  config.yaml: |
    model_list:
      - model_name: gpt-4o
        litellm_params:
          model: openai/gpt-4o
          api_key: os.environ/OPENAI_API_KEY
      - model_name: claude-sonnet
        litellm_params:
          model: anthropic/claude-sonnet-4-20250514
          api_key: os.environ/ANTHROPIC_API_KEY
    general_settings:
      master_key: os.environ/LITELLM_MASTER_KEY
    litellm_settings:
      num_retries: 3
      fallbacks:
        - claude-sonnet: [gpt-4o]
✓ AgentGateway ConfigMap
apiVersion: v1
kind: ConfigMap
metadata:
  name: agentgateway-config
data:
  config.yaml: |
    llm:
      models:
        - name: gpt-4o
          provider: openAI
          params:
            model: gpt-4o
            apiKey: "$OPENAI_API_KEY"
        - name: claude-sonnet
          provider: anthropic
          params:
            model: claude-sonnet-4-20250514
            apiKey: "$ANTHROPIC_API_KEY"

Deployment

✕ LiteLLM Deployment
apiVersion: apps/v1
kind: Deployment
metadata:
  name: litellm
spec:
  replicas: 2
  selector:
    matchLabels:
      app: litellm
  template:
    spec:
      containers:
      - name: litellm
        image: docker.litellm.ai/berriai/litellm:main-stable
        args: ["--config", "/app/config.yaml", "--port", "4000"]
        ports:
        - containerPort: 4000
        envFrom:
        - secretRef:
            name: litellm-secrets
        volumeMounts:
        - name: config
          mountPath: /app/config.yaml
          subPath: config.yaml
        livenessProbe:
          httpGet:
            path: /health/liveliness
            port: 4000
          initialDelaySeconds: 120
        readinessProbe:
          httpGet:
            path: /health/readiness
            port: 4000
          initialDelaySeconds: 120
        resources:
          requests:
            cpu: "2"
            memory: 4Gi
          limits:
            cpu: "4"
            memory: 8Gi
      volumes:
      - name: config
        configMap:
          name: litellm-config

        ports:
        - containerPort: 4000
          name: llm
        - containerPort: 15000
          name: admin
        envFrom:
        - secretRef:
            name: agentgateway-secrets

        livenessProbe:
          httpGet:
            path: /healthz
            port: 15000

        readinessProbe:
          httpGet:
            path: /healthz
            port: 15000
          initialDelaySeconds: 5

Option 2: Helm + Gateway API (Recommended)

For production Kubernetes deployments, AgentGateway integrates with the Kubernetes Gateway API through its control plane (kgateway).

Install AgentGateway with Helm

# Add the Helm repository
helm repo add agentgateway https://agentgateway.dev/charts
helm repo update

# Install the control plane + data plane
helm install agentgateway agentgateway/agentgateway \
  --namespace agentgateway-system \
  --create-namespace

Create a Gateway resource

apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: ai-gateway
  namespace: default
spec:
  gatewayClassName: agentgateway
  listeners:
  - name: llm
    port: 4000
    protocol: HTTP

Define backends with CRDs

apiVersion: agentgateway.dev/v1alpha1
kind: AgentgatewayBackend
metadata:
  name: openai
spec:
  ai:
    provider:
      openai:
        model: "gpt-4o"
  policies:
    auth:
      secretRef:
        name: openai-secret
---
apiVersion: agentgateway.dev/v1alpha1
kind: AgentgatewayBackend
metadata:
  name: anthropic
spec:
  ai:
    provider:
      anthropic:
        model: "claude-sonnet-4-20250514"
  policies:
    auth:
      secretRef:
        name: anthropic-secret

Create HTTPRoute to wire it together

apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: llm-route
spec:
  parentRefs:
  - name: ai-gateway
  rules:
  - backendRefs:
    - group: agentgateway.dev
      kind: AgentgatewayBackend
      name: openai
    - group: agentgateway.dev
      kind: AgentgatewayBackend
      name: anthropic
Why Gateway API? Using Gateway API CRDs means your AI gateway integrates with the same infrastructure tooling as your HTTP gateways. Config changes are dynamic (xDS push), no restart needed. You also get RBAC, multi-tenancy, and fleet management built in.

K8s Migration Checklist

Translate ConfigMap contents

Convert your LiteLLM config.yaml to AgentGateway format (see LLM Gateway and MCP Gateway pages).

Update Secrets

Remove LITELLM_MASTER_KEY. Keep OPENAI_API_KEY and ANTHROPIC_API_KEY (same names work).

Swap the Deployment

Change image, args, resource limits, and probes. Drop initialDelaySeconds from 120s to 5s.

Update Service

Add admin port (15000) if you want the built-in UI. LLM port stays at 4000.

Verify

Run the same curl test against the Service. No client changes needed.

kubectl port-forward svc/agentgateway 4000:4000 &

curl http://localhost:4000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello!"}]}'
