Binary & Docker Deployment

Replace your LiteLLM Python process with a single AgentGateway binary. No pip, no virtualenv, no Python runtime.

Option 1: Binary Install

Before: LiteLLM

# Install LiteLLM
pip install 'litellm[proxy]'

# Run the proxy
litellm --config config.yaml --port 4000

After: AgentGateway

# Install AgentGateway (one command, no dependencies)
curl -sL https://agentgateway.dev/install | bash

# Run the gateway
agentgateway -f config.yaml
That's it. No Python, no pip, no virtual environment, no dependency conflicts. One static binary for macOS (Intel/ARM), Linux (x86/ARM), and Windows.

Install a Specific Version

curl -sL https://agentgateway.dev/install | bash -s -- --version 1.0.0

Option 2: Docker

Before: LiteLLM Docker

docker run \
  -v $(pwd)/config.yaml:/app/config.yaml \
  -e OPENAI_API_KEY=$OPENAI_API_KEY \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  -p 4000:4000 \
  docker.litellm.ai/berriai/litellm:main-latest \
  --config /app/config.yaml

After: AgentGateway Docker

docker run \
  -v $(pwd)/config.yaml:/config.yaml \
  -e OPENAI_API_KEY=$OPENAI_API_KEY \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  -e ADMIN_ADDR=0.0.0.0:15000 \
  -p 4000:4000 \
  -p 127.0.0.1:15000:15000 \
  cr.agentgateway.dev/agentgateway:latest \
  -f /config.yaml
Admin UI included Port 15000 gives you a built-in web UI for inspecting routes, backends, and live config. Access it at http://localhost:15000/ui.

Quick Start: 5-Minute Demo

Go from zero to working LLM + MCP gateway in under 5 minutes.

Install AgentGateway

curl -sL https://agentgateway.dev/install | bash

Set your API keys

export OPENAI_API_KEY="sk-your-key-here"
export ANTHROPIC_API_KEY="sk-ant-your-key-here"

Create config.yaml

cat <<'EOF' > config.yaml
# LLM Gateway on port 4000
llm:
  models:
    - name: gpt-4o
      provider: openAI
      params:
        model: gpt-4o
        apiKey: "$OPENAI_API_KEY"
    - name: claude-sonnet
      provider: anthropic
      params:
        model: claude-sonnet-4-20250514
        apiKey: "$ANTHROPIC_API_KEY"
EOF

Run it

agentgateway -f config.yaml

Test it

# Test OpenAI model
curl http://localhost:4000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello!"}]}'

# Test Anthropic model (same endpoint!)
curl http://localhost:4000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "claude-sonnet", "messages": [{"role": "user", "content": "Hello!"}]}'

# Open Admin UI
open http://localhost:15000/ui

Full Demo: LLM + MCP Gateway

Run both LLM proxy and MCP gateway in a single process.

Create the combined config

cat <<'EOF' > config.yaml
binds:
# LLM Gateway
- port: 4000
  listeners:
  - name: llm
    protocol: HTTP
    routes:
    - name: models
      backends:
      - ai:
          groups:
          - providers:
            - name: gpt-4o
              provider:
                openAI:
                  model: gpt-4o
                  apiKey: "$OPENAI_API_KEY"
            - name: claude-sonnet
              provider:
                anthropic:
                  model: claude-sonnet-4-20250514
                  apiKey: "$ANTHROPIC_API_KEY"

# MCP Gateway
- port: 3000
  listeners:
  - name: mcp
    protocol: HTTP
    routes:
    - name: tools
      backends:
      - mcp:
          targets:
          - name: everything
            stdio:
              cmd: npx
              args: ["-y", "@modelcontextprotocol/server-everything"]
EOF

Run it

agentgateway -f config.yaml

You now have:

  • LLM Gateway at http://localhost:4000 (OpenAI-compatible API)
  • MCP Gateway at http://localhost:3000 (MCP protocol)
  • Admin UI at http://localhost:15000/ui

Point your Python code to it

# No code changes needed if you're already using OpenAI SDK
import openai

client = openai.OpenAI(
    api_key="anything",  # auth handled by gateway
    base_url="http://localhost:4000/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",  # or "claude-sonnet"
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)

Hot Reload (File Watch)

AgentGateway watches the config file and applies changes without restart. Edit config.yaml while the gateway is running -- changes take effect immediately.

# Edit config while running (no restart needed)
vim config.yaml

# AgentGateway detects the change and reloads automatically
# Check the admin UI to verify: http://localhost:15000/ui
Zero downtime config changes LiteLLM requires a process restart for config changes. AgentGateway watches the config file and applies changes live. In Kubernetes, this works with xDS for cluster-wide dynamic config.
